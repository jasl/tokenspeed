# DeepSeek V4 SM12x PoC Implementation Plan

**Goal:** Build a correctness-first DeepSeek V4 SM12x PoC in TokenSpeed, reusing the proven vLLM SM12x operator ideas without importing vLLM runner architecture.

**Architecture:** Treat SM120/SM121 as explicit TokenSpeed targets, add portable fallback operators behind TokenSpeed's kernel boundaries, and route only DeepSeek V4 eager attention/indexer paths through them. Keep MTP, CUDA graph capture, and pipeline parallelism out of the first PoC.

**Tech Stack:** Python, PyTorch, TokenSpeed runtime, tokenspeed-kernel, Triton via `tokenspeed_kernel._triton`, CUDA SM120/SM121, existing `ds4-sm120-harness` OpenAI-compatible correctness checks.
